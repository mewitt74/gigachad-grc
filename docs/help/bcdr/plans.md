# BC/DR Plans

Recovery plans document how your organization will respond to and recover from disruptions.

## Overview

Plan management helps you:
- Document recovery procedures
- Maintain current contact information
- Track plan reviews and updates
- Link to processes and resources
- Support DR testing

## Plan Types

| Type | Purpose |
|------|---------|
| **Business Continuity Plan** | Overall BC strategy and coordination |
| **Disaster Recovery Plan** | IT system recovery |
| **Crisis Management Plan** | Executive-level response |
| **Department Plan** | Department-specific procedures |
| **Application Recovery Plan** | Specific application recovery |

## Creating a Plan

1. Click **Create Plan**
2. Enter:
   - **Name**: Plan title
   - **Type**: Plan category
   - **Owner**: Responsible person
   - **Description**: Plan scope
3. Click **Create**

## Plan Structure

### Plan Content

#### 1. Purpose & Scope
- What the plan covers
- When it should be invoked
- Out-of-scope items

#### 2. Roles & Responsibilities
- Recovery team structure
- Role definitions
- Decision authority

#### 3. Activation Criteria
- What triggers activation
- Who can activate
- Escalation thresholds

#### 4. Response Procedures
- Immediate actions
- Assessment steps
- Communication requirements

#### 5. Recovery Procedures
- Step-by-step recovery
- Resource requirements
- Dependencies

#### 6. Return to Normal
- Criteria for return
- Transition steps
- Post-incident review

### Contact Information
Maintain current contacts:
- Team members with roles
- Escalation contacts
- Vendor contacts
- Emergency services

### Resource Requirements
Document needed resources:
- Personnel
- Equipment
- Facilities
- Third-party services

## Plan Editor

### Rich Text Editor
Write plan content with:
- Headings and sections
- Bullet and numbered lists
- Tables
- Images
- Links

### Document Upload
Alternatively:
1. Upload existing plan document (Word, PDF)
2. Link to external document
3. Reference shared drive location

### Checklists
Create actionable checklists:
1. Go to **Checklists** tab
2. Add checklist sections
3. Add items with assignees
4. Use during activation

## Version Control

### Version History
Track all plan changes:
- Version number
- Change date
- Changed by
- Change summary

### Creating New Version
1. Open plan
2. Click **Create New Version**
3. Make updates
4. Save with version notes

### Comparing Versions
1. Go to **Version History**
2. Select two versions
3. View side-by-side comparison

## Plan Review

### Review Schedule
Set review frequency:
- Annual (minimum)
- Semi-annual (recommended for critical)
- After significant changes
- Post-incident

### Review Process
1. Owner receives review reminder
2. Review plan content for accuracy
3. Update outdated information
4. Verify contact information
5. Mark review complete

### Review Workflow
1. **Due for Review**: Approaching review date
2. **Under Review**: Review in progress
3. **Reviewed**: Review complete
4. **Overdue**: Past review date

## Linking

### To Business Processes
Link plan to processes it covers:
1. Go to **Processes** tab
2. Click **Link Process**
3. Select processes
4. Document coverage

### To Assets
Link critical assets:
1. Go to **Assets** tab
2. Link IT systems, applications
3. Note recovery details

### To Vendors
Link critical vendors:
1. Go to **Vendors** tab
2. Link vendor records
3. Include vendor recovery contacts

## Testing

### Test Documentation
Link plan to DR tests:
1. Go to **Tests** tab
2. See tests that exercised this plan
3. View test results

### Lessons Learned
After tests or real incidents:
1. Document lessons learned
2. Update plan as needed
3. Track improvement actions

## Distribution

### Distribute Plan
Share plan with stakeholders:
1. Click **Distribute**
2. Select recipients
3. Choose format (PDF, link)
4. Send notification

### Access Control
Control who can:
- View the plan
- Edit the plan
- Approve changes

## Best Practices

### Plan Development
- Keep it practical and usable
- Include clear step-by-step procedures
- Test and validate procedures
- Keep it current

### Maintenance
- Regular reviews (at least annual)
- Update after organizational changes
- Update after incidents/tests
- Version control all changes

### Accessibility
- Store in accessible location
- Maintain offline copies
- Distribute to key personnel
- Test access during exercises
