# Communication Plans

Communication plans ensure effective stakeholder communication during incidents and recovery activities.

## Overview

Communication plans help you:
- Notify stakeholders quickly
- Provide consistent messaging
- Meet notification requirements
- Coordinate response teams
- Keep customers informed

## Plan Components

### Stakeholder Groups
Define who needs to be communicated with:
- Executive leadership
- IT teams
- Business units
- Customers
- Partners/vendors
- Regulators
- Media (if applicable)
- Employees

### Contact Information
Maintain current contacts:
- Names and roles
- Phone numbers
- Email addresses
- Alternate contacts
- Escalation paths

### Communication Channels
How to reach stakeholders:
- Phone trees
- SMS/text alerts
- Email
- Collaboration tools (Slack, Teams)
- Mass notification system
- Status page
- Social media

   - **Name**: Plan title
   - **Type**: Crisis, Incident, Stakeholder
   - **Owner**: Plan owner
4. Define stakeholder groups
5. Add contacts
6. Create message templates
7. Click **Save**

## Message Templates

### Pre-Written Messages
Prepare messages in advance:
- Initial notification
- Status updates
- Resolution notification
- Post-incident summary

### Template Variables
Use placeholders:
- `{{incident_type}}`
- `{{affected_systems}}`
- `{{expected_resolution}}`
- `{{contact_info}}`

### Creating Templates
1. Go to **Templates** tab
2. Click **Add Template**
3. Select message type
4. Write template content
5. Add variables
6. Save

## Notification Workflow

### Triggering Communication
1. Incident occurs
2. Communication plan activated
3. Select appropriate stakeholder groups
4. Choose message template
5. Customize as needed
6. Send notification

### Escalation
Define escalation procedures:
- Time-based escalation
- Severity-based escalation
- No-response escalation

### Acknowledgment
Track who received messages:
- Delivery confirmation
- Read receipts (if available)
- Response required

## Status Page

### Public Status Updates
If enabled, publish to status page:
1. Go to **Status Page** tab
2. Create incident notice
3. Post updates
4. Resolve when complete

### Customer Communication
Customer-facing updates:
- Simplified messaging
- No technical jargon
- Clear impact statement
- Expected resolution

## Integration

### Mass Notification Systems
Integrate with:
- PagerDuty
- Everbridge

- OpsGenie

### Collaboration Tools
Push to:
- Slack channels
- Microsoft Teams
- Email lists

## Testing Communication

### Contact Verification
Regularly verify:
- Contact info is current
- Numbers work
- Email delivery
- Escalation paths

### Communication Drill
Test the plan:
1. Schedule drill
2. Simulate incident
3. Activate communication plan
4. Track responses
5. Measure timing
6. Document issues

## Best Practices

### Maintenance
- Review contacts quarterly
- Update after personnel changes
- Test communication channels
- Keep templates current

### During Incidents
- Communicate early
- Provide regular updates
- Be honest about unknowns
- Give next update time

### Post-Incident
- Send resolution notice
- Thank stakeholders
- Share lessons learned
- Request feedback
