# Compliance Calendar

The Compliance Calendar helps you track and manage deadlines, reviews, audits, and other time-sensitive compliance activities.

## Overview

The calendar provides:
- Visual timeline of compliance deadlines
- Scheduled control testing dates
- Policy review reminders
- Audit schedules
- Evidence expiration alerts

## Calendar Views

### Month View
Default view showing a monthly calendar with:
- Color-coded event types
- Quick event preview on hover
- Click to view full details

### Week View
Detailed weekly view for busy periods:
- Hour-by-hour breakdown
- Multiple events per day
- Better for scheduling meetings

### List View
Chronological list of upcoming events:
- Best for review and planning
- Filter and sort by type
- Export to spreadsheet

### Agenda View
Today's and upcoming items:
- Focus on what's due soon
- Quick action buttons
- Priority indicators

## Event Types

| Type | Color | Description |
|------|-------|-------------|
| **Control Test** | Blue | Scheduled control testing |
| **Policy Review** | Purple | Policy due for review |
| **Audit** | Red | Internal or external audit |
| **Evidence Refresh** | Orange | Evidence expiring soon |
| **Framework Assessment** | Green | Framework readiness check |
| **Custom Event** | Gray | User-created events |

## Creating Events

### Quick Add
1. Click any date on the calendar
2. Enter event title
3. Select event type
4. Add quick details
5. Save

### Detailed Event
1. Click **Add Event**
2. Fill in all fields:
   - **Title**: Event name
   - **Type**: Event category
   - **Date/Time**: When it occurs
   - **Duration**: How long
   - **Assignee**: Who's responsible
   - **Description**: Details
   - **Related Items**: Controls, policies, etc.
3. Set reminders
4. Save

### Recurring Events
1. Create event as usual
2. Toggle **Repeat** on
3. Configure recurrence:
   - Daily, Weekly, Monthly, Yearly
   - Every X intervals
   - End date or number of occurrences
4. Save

## Automated Events

The calendar automatically shows:

### Control Testing
- Tests scheduled from the Controls module
- Shows control ID and test type
- Click to view control details

### Policy Reviews
- Reviews due based on policy schedules
- Shows days until due
- Click to start review

### Evidence Expiration
- Evidence approaching expiration
- Shows days remaining
- Click to refresh evidence

### Audit Milestones
- Audit start and end dates
- Request deadlines
- Finding remediation due dates

## Reminders

### Configure Reminders
1. Open any event
2. Click **Add Reminder**
3. Set reminder timing:
   - 1 week before
   - 3 days before
   - 1 day before
   - Day of (morning)
   - Custom

### Notification Delivery
Reminders are sent via:
- In-app notifications
- Email (if configured)
- Dashboard alerts

### Personal Preferences
Configure your defaults in **Account Settings → Notifications**:
- Default reminder timing
- Delivery method preferences
- Quiet hours

## Filtering

### By Event Type
Check/uncheck event types to show/hide:
- ☑ Control Tests
- ☑ Policy Reviews
- ☑ Audits
- ☑ Evidence Refresh
- ☐ Custom Events

### By Assignee
Filter to show only:
- My events
- My team's events
- All events

### By Status
- Upcoming
- Overdue
- Completed

## Integration

### External Calendars
Export events to external calendars:
1. Click **Calendar Settings**
2. Select **Export**
3. Choose format:
   - iCal (.ics) for any calendar app
   - Google Calendar direct link
   - Outlook integration
4. Subscribe or download

### Sync Options
- **One-time export**: Download current events
- **Subscribe**: Auto-updating calendar feed
- **Two-way sync**: Changes sync both ways (if supported)

## Reporting

### Calendar Summary
Generate reports on:
- Upcoming events this week/month
- Overdue items
- Completion rates
- Workload by assignee

### Export Calendar
1. Click **Export**
2. Select date range
3. Choose format (PDF, CSV, Excel)
4. Download

## Best Practices

### Planning
- Review calendar weekly
- Plan capacity for busy periods
- Coordinate audit schedules early

### Prioritization
- Focus on overdue items first
- Group similar tasks
- Delegate when overloaded

### Documentation
- Record outcomes in related items
- Update statuses after completion
- Note any rescheduled items
