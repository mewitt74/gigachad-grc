# Export Options

Export your data from GigaChad GRC for external use, backup, or analysis.

## Overview

Export capabilities allow you to:
- Download data in multiple formats
- Create backups
- Share with stakeholders
- Analyze in external tools
- Meet compliance requirements

## Export Formats

| Format | Description | Best For |
|--------|-------------|----------|
| **CSV** | Comma-separated values | Data analysis, Excel import |
| **Excel** | Native Excel format | Reporting, analysis |
| **PDF** | Formatted document | Sharing, printing, audits |
| **JSON** | Structured data | System integration, backup |
| **Word** | Editable document | Policy documents |

## Exporting from Lists

### Quick Export
From any list view:
1. Apply desired filters
2. Click **Export**
3. Select format
4. Download

### Selective Export
Export specific items:
1. Select items (checkboxes)
2. Click **Export Selected**
3. Choose format
4. Download

### Export All
Export complete data:
1. Clear filters (or keep desired)
2. Click **Export**
3. Select **Export All**
4. Choose format

## Organization Data Export

### Full Data Export
Export all organization data:
1. Go to **Settings → Organization**
2. Find **Data Portability**
3. Click **Export Organization Data (JSON)**
4. Download comprehensive backup

### What's Included
- Risks and assessments
- Controls and implementations
- Evidence metadata
- Vendors and contracts
- Audits and findings
- Audit logs (configurable)

### Use Cases
- System migration
- Compliance documentation
- Business continuity backup
- Offboarding preparation

## Scheduled Exports

### Automatic Exports
Configure regular exports:
1. Go to **Tools → Scheduled Reports**
2. Create export schedule
3. Select data to export
4. Set frequency
5. Configure delivery

### Delivery Options
- Email attachment
- Cloud storage upload
- SFTP transfer
- Webhook notification

## Export Permissions

### Access Control
Export permissions based on:
- User role

- Module access

### Audit Trail
All exports logged:
- Who exported

- Format

## Data Considerations

### Sensitive Data
Exports may contain:
- Personal information
- Security details
- Business data

Handle appropriately:
- Secure transmission
- Encrypted storage
- Access controls

### Data Size
Large exports:
- May take time to generate
- Download may timeout
- Consider filtering

## Best Practices

### Before Exporting
- Define purpose
- Apply appropriate filters
- Consider data sensitivity
- Choose right format

### After Exporting
- Store securely
- Use encryption if needed
- Document retention
- Delete when no longer needed

### Compliance
- Know your requirements
- Document exports
- Secure transmission
- Proper retention

## Troubleshooting

### Export Fails
1. Reduce data size (filters)
2. Try different format
3. Check permissions
4. Contact support

### Incomplete Data
1. Check filters
2. Verify permissions
3. Review date range
4. Check export settings

### File Won't Open
1. Verify correct format
2. Try different application
3. Check file integrity
4. Re-export
