# Audit Requests

Audit requests track evidence and information needed during an audit. This guide covers creating, managing, and fulfilling audit requests.

## Overview

Requests help you:
- Track what auditors need
- Assign evidence collection tasks
- Monitor completion status
- Meet audit deadlines

## Viewing Requests

Navigate to **Audit → Audit Requests** to see all requests.

### Filters
- **Audit**: Filter by specific audit
- **Status**: Open, In Progress, etc.
- **Assigned To**: Your requests
- **Due Date**: Overdue, due this week

### My Queue
View requests assigned to you:
1. Go to **Audit → Audit Requests**
2. Click **My Requests** filter

## Creating Requests

### Single Request
1. Open an audit
2. Go to **Requests** tab
3. Click **Create Request**
4. Fill in:
   - **Title**: Brief description
   - **Description**: Detailed requirements
   - **Control**: Related control (if any)
   - **Assigned To**: Who should provide
   - **Due Date**: When needed
5. Click **Create**

### Bulk Create
1. Click **Create Request** → **Bulk Create**
2. Download CSV template
3. Fill in request details
4. Upload and review
5. Create all requests

## Fulfilling Requests

### As the Assignee
1. Find your assigned request
2. Click to open details
3. Review what's needed
4. Gather evidence
5. Upload files or link existing evidence
6. Add notes explaining the evidence
7. Click **Submit for Review**

### Uploading Evidence
- Drag and drop files
- Or click to browse
- Add description for each file
- Link to existing evidence items

## Request Review

### As the Reviewer
1. Open submitted request
2. Review provided evidence
3. Check against requirements
4. Choose action:
   - **Accept**: Evidence is sufficient
   - **Reject**: Need more/different evidence
5. Add review notes

### Rejection Handling
If rejected:
1. Review rejection notes
2. Gather additional evidence
3. Re-submit request

## Request Details

Click any request to see:

### Information
- Request details
- Assignment and dates
- Related control/framework

### Evidence
- Uploaded files
- Linked evidence items
- Evidence history

### Activity
- Status changes
- Comments
- Review history

## Communication

### Comments
Add comments to discuss requests:
1. Open request detail
2. Scroll to **Comments**
3. Type your message
4. Click **Post**

### Notifications
Automatic notifications for:
- New assignment
- Due date approaching
- Status changes
- New comments

## Bulk Actions

### Select Multiple
1. Check boxes on request list
2. Click **Bulk Actions**
3. Choose action:
   - Reassign
   - Change due date
   - Mark as in progress

## Reporting

### Request Status Report
Shows:
- Total requests by status
- Overdue requests
- Completion rate
- Average response time

### Export
Export request data:
1. Apply filters
2. Click **Export**
3. Download CSV

## Best Practices

### Creating Requests
- Be specific about what's needed
- Include examples if helpful
- Set realistic due dates
- Assign to the right person

### Fulfilling Requests
- Read requirements carefully
- Provide context with evidence
- Submit before due date
- Ask questions if unclear

### Managing Requests
- Review status daily
- Follow up on overdue
- Re-assign if needed
- Communicate blockers
