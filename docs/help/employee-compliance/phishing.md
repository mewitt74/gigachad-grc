# Phishing Simulations

Test your organization's security awareness with realistic phishing simulations.

## Overview

Phishing simulations help:
- Assess employee susceptibility to phishing attacks
- Identify users who need additional training
- Track improvements over time
- Meet compliance requirements for security awareness

## Creating a Campaign

### 1. Navigate to Campaigns

Go to **Tools → Phishing Simulations → Campaigns**

### 2. Click Create Campaign

Provide campaign details:

| Field | Description |
|-------|-------------|
| **Name** | Campaign identifier |
| **Description** | Internal notes about the campaign |
| **Template** | Pre-built or custom phishing email |
| **Target Groups** | User groups to receive the simulation |
| **Schedule** | When to send (immediate, scheduled, staggered) |

### 3. Select Template

Choose from template categories:

- **Credential Harvesting**: Fake login pages
- **Malware**: Attachment or download links
- **Gift Card Scams**: Urgency-based requests
- **IT Support**: Password reset requests
- **Executive Impersonation**: CEO fraud scenarios

### 4. Configure Landing Page

Set up what users see when they click:

- **Training Redirect**: Immediate training opportunity
- **Awareness Message**: Educational content
- **Tracking Only**: Silent tracking without notification

### 5. Set Schedule

Scheduling options:

- **Immediate**: Send to all targets now
- **Scheduled**: Set specific date/time
- **Staggered**: Spread over hours/days to avoid detection

## Campaign Analytics

### Real-Time Tracking

Monitor as the campaign runs:

- **Sent**: Emails delivered
- **Opened**: Users who opened the email
- **Clicked**: Users who clicked links
- **Reported**: Users who reported to IT
- **Entered Data**: Users who submitted credentials

### Department Breakdown

View results by department:
- Click rates by team
- Report rates by team
- Compare performance across organization

### User Risk Scores

Individual user metrics:
- Historical click rates
- Training completion
- Report rates
- Risk level (High, Medium, Low)

## Template Library

### Pre-Built Templates

| Category | Templates |
|----------|-----------|
| IT/Technical | Password reset, MFA setup, Account locked |
| HR | Benefits enrollment, Policy update, Performance review |
| Finance | Invoice attached, Payment required, Expense report |
| Executive | CEO request, Confidential matter, Urgent transfer |
| External | Shipping notification, Social media alert, Subscription |

### Custom Templates

Create your own templates:

1. Go to **Templates → Create New**
2. Design email content (HTML supported)
3. Configure landing page
4. Set difficulty rating
5. Save and test

### Template Best Practices

- Match your organization's email style
- Use realistic sender addresses
- Include believable urgency
- Test on small groups first

## Reporting Users

### Report Button Integration

Train users to report suspicious emails:

1. Configure report button in email client
2. Track reports in analytics
3. Reward reporting behavior

### Report Analysis

When users report:
- Simulation marked as reported
- User receives positive feedback
- Metric tracked in analytics

## Post-Campaign Actions

### Automatic Training Assignment

Configure automatic responses:

| Action | Trigger |
|--------|---------|
| Assign training | User clicks link |
| Send reminder | User enters data |
| Manager notification | Repeat offenders |

### Follow-Up Training

Targeted training based on behavior:
- Basic awareness for first-time clickers
- Advanced training for repeat offenders
- Recognition for reporters

## Compliance & Reporting

### Audit Evidence

Generate reports for auditors:
- Campaign summaries
- Training completion rates
- Improvement trends
- Department comparisons

### Export Options

## Best Practices

### Campaign Frequency

- Monthly simulations recommended
- Vary templates and timing
- Track trends over quarters

### Difficulty Progression

- Start with obvious attempts
- Gradually increase sophistication
- Match real-world threats

### Positive Reinforcement

- Recognize good behavior
- Celebrate reporters
- Avoid punitive approaches
- Focus on learning

### Privacy Considerations

- Follow HR policies
- Consider union requirements
- Document purpose clearly
- Maintain confidentiality

## Troubleshooting

### Low Open Rates

- Check spam filters
- Verify email delivery
- Review subject lines

### High Click Rates

- Templates may be too convincing
- Consider additional training
- Adjust difficulty level

### Reports Not Tracking

- Verify report button configuration
- Check integration settings
- Test with known simulation
